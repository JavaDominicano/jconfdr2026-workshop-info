### Requirements for Attendees

Technical:

-   A laptop with JDK 21 or later installed. Ideally, you have admin rights on the laptop, so you can install additional things, but we'll manage with what we have.
-   An IDE of your choice (I will be using IntelliJ IDEA (Community Edition is fine), so it might be easier to follow if you have it, but it's not a requirement).
-   Git and Maven
-   I will provide the frontend for the API that we will build, and you'll need npm (Node.js 20 LTS or 22 LTS) to use that. This is not a hard requirement for the course, just means you won't be able to use the UI. The workshop is primarily about API building and backend development, so we will not be talking about the frontend; it's just sometimes nice to see how that API could be used.
-   All your energy and curiosity – We'll be making APIs fun, and trust me, you'll enjoy every bit of it!

Knowledge:

-   Comfortable writing and running a basic Spring Boot application (controllers, beans, dependency injection).
-   General proficiency in Java, Maven, and Git (nothing advanced; you're required to know how to clone a project, switch branches, and build using Maven).
-   No prior GraphQL experience required.

People are still welcome to join the workshop without any or all of the above; that just means they won't be able to do the hands-on work, but will still definitely learn something.

### Bonus (Extra)
We live in an age of AI, so after we complete everything, if there's time and interest among students, we will try (or at least I will demonstrate how) to expose our newly built API to the AI, explaining and demonstrating the hottest topic at the moment - APIs for AIs - this is one of the hottest use cases for GraphQL at the moment.
