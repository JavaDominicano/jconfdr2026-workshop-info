### Requisitos para los asistentes

#### Técnicos:
- Una laptop con JDK 21 o superior instalado. Idealmente, con derechos de administrador para instalar herramientas adicionales, pero trabajaremos con lo que tengamos.
- Un IDE de tu preferencia (yo usaré IntelliJ IDEA, la edición Community es suficiente). No es obligatorio, pero facilitará el seguimiento.
- Git y Maven instalados.
- Yo proporcionaré el frontend para la API que construiremos. Para usarlo, necesitarás **npm (Node.js 20 LTS o 22 LTS)**. *No es un requisito obligatorio para el taller*, pero sin él no podrás interactuar con la interfaz. El enfoque principal será el **desarrollo de la API y el backend**, así que no profundizaremos en el frontend.
- ¡Y lo más importante! Toda tu **energía y curiosidad**. Haremos que las APIs sean divertidas, ¡y te aseguro que disfrutarás cada momento!

#### Conocimientos previos:
-  **Experiencia básica en Spring Boot**: Saber crear y ejecutar una aplicación sencilla (controladores, beans, inyección de dependencias).
-  **Conocimientos generales de Java, Maven y Git**: No es necesario ser experto, pero sí saber cómo clonar un proyecto, cambiar de rama y compilar con Maven.
-  **No se requiere experiencia previa en GraphQL.**

💡 **Nota:** Todos son bienvenidos, incluso sin cumplir todos los requisitos. Quienes no puedan realizar las actividades prácticas **igualmente aprenderán algo nuevo**.

**Vivimos en la era de la IA**, así que, una vez que completemos todo, si hay tiempo e interés entre los estudiantes, intentaremos (o al menos yo demostraré cómo) **exponer nuestra API recién creada a la IA**, explicando y mostrando el tema más candente del momento: **APIs para IAs**, uno de los casos de uso más relevantes para GraphQL actualmente.
